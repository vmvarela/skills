# Contributing to snapenv

We welcome contributions from the community!

## Code of Conduct

Our community follows a Code of Conduct to ensure a safe and welcoming environment for all contributors.

## How to Contribute

### Reporting Bugs

If you find a bug, please report it by creating an issue on GitHub. Include:
- Description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior

### Suggesting Features

We welcome feature suggestions. Please open an issue with:
- Description of the feature
- Use cases
- Why this feature would be useful

### Contributing Code

#### Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/snapenv.git`
3. Create a feature branch: `git checkout -b feature/your-feature`
4. Make your changes
5. Test your changes: `go test ./...`
6. Commit your changes with clear messages
7. Push to your fork
8. Create a Pull Request

#### Code Style

- Follow Go conventions
- Run `go fmt` on your code
- Use meaningful variable names
- Add comments for complex logic

#### Testing

All code must be tested:
```bash
go test ./...
```

## Pull Request Process

1. Update documentation as needed
2. Ensure all tests pass
3. Provide a clear PR description
4. Link related issues
